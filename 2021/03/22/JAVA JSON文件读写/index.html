excutor

一个设计好的线程池框架

本来我自己想用普通的Thread的线程，实现6个线程计算10到1000的累加

然后怎么算都是错的

我放弃了

然后看到了excutor框架

第一个是确定线程池里最多有几个线程

第二个是存放一个result的数组

excutor和Runnable的区别就在于excutor需要覆盖的call方法可以有返回值，而run不需要。他们作为方法一个由excutor.submit提交就可以运行线程。而run由线程start开始运行。

这样就能把结果存进数组里，然后要得到最终结果只需要遍历数组就可以了。
